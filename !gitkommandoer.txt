Hente nytt remote repo fra Git (sjelden):
git clone <url>

Hente data fra remote repo til eksisterende prosjekt (gjør hver gang før du begynner å jobbe):
git pull

Sjekke status lokalt:
git status

Sjekke inn filer lokalt: Viktig: Dette gjør kun en sjekk inn på ditt lokale filsystem. Fordelen er at det er veldig lett å rulle tilbake hvis det trengs, ulempen er at du må huske å sende filene til remote repo som en egen kommando.
git commit -a -m'Melding'

Legge til ny fil som ennå ikke er versjonskontrollert:
git add <fil>

Sende alle commits du har gjort til remote repo (gjøres enten etter hver commit, eller når du er ferdig for dagen):
git push

Ignorere fil fra git:
Legg inn filnavn i .gitignore

Hvis jeg har glemt å gjøre en pull før jeg endrer filer?
Da funker ikke push. Gjør da først:
git pull --rebase
Og gjør en push igjen

Dersom du har endret "fil.txt" på NIH, pushet, reist hjem og endrer "fil.txt" hjemme uten å gjøre en pull først, vil git etter beste evne forsøke å merge filen når du endelig gjør en pull. Hvis dette ikke går, vil git lage en konflikt. Da må du manuelt rette opp i filen før den kan pushes. Søk på resolve conflicts in git så finner du oppskrift.
